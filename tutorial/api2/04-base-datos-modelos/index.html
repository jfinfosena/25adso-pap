    4. Base de Datos y Modelos

    ¬øQu√© es SQLAlchemy?

    SQLAlchemy es un ORM (Object-Relational Mapping) que nos permite: - Trabajar con bases de datos usando objetos Python - Escribir menos SQL manual - Cambiar de base de datos f√°cilmente - Tener mayor seguridad contra inyecciones SQL

    ORM vs SQL Tradicional

    SQL Tradicional: 

    CREATE TABLE users (
    id INTEGER PRIMARY KEY,
    name VARCHAR(100),
    email VARCHAR(100)
);

INSERT INTO users (name, email) VALUES ('Juan', 'juan@email.com');
SELECT * FROM users WHERE id = 1;

    Con SQLAlchemy ORM: 

    class User(Base):
    __tablename__ = "users"
    id = Column(Integer, primary_key=True)
    name = Column(String)
    email = Column(String)

# Crear usuario
user = User(name="Juan", email="juan@email.com")
db.add(user)
db.commit()

# Obtener usuario
user = db.query(User).filter(User.id == 1).first()

    Configuraci√≥n de la Base de Datos

    Paso 1: Crear el archivo de configuraci√≥n

    Creamos app/database/database.py:

    from sqlalchemy import create_engine
from sqlalchemy.ext.declarative import declarative_base
from sqlalchemy.orm import sessionmaker

# URL de conexi√≥n a la base de datos

# Crear la clase de sesi√≥n

# Crear la clase base para los modelos

# Funci√≥n para obtener una sesi√≥n de base de datos

    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()

    Explicaci√≥n Detallada

    üîß create_engine()

    engine = create_engine(SQLALCHEMY_DATABASE_URL)
    - Prop√≥sito: Crea la conexi√≥n principal a la base de datos - SQLite: Base de datos en archivo local (ideal para desarrollo) - Archivo: test.db se crea autom√°ticamente

    üìù sessionmaker()

    SessionLocal = sessionmaker(autocommit=False, autoflush=False, bind=engine)
    - autocommit=False: Las transacciones deben confirmarse manualmente - autoflush=False: Los cambios no se env√≠an autom√°ticamente - bind=engine: Vincula las sesiones al motor de base de datos

    üèóÔ∏è declarative_base()

    Base = declarative_base()
    - Prop√≥sito: Clase base para todos nuestros modelos - Funcionalidad: Proporciona metadatos y funciones ORM

    üîÑ get_db()

    def get_db():
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()
    - Patr√≥n: Dependency Injection de FastAPI - yield: Genera una sesi√≥n y la mantiene abierta - finally: Garantiza que la sesi√≥n se cierre

    Anatom√≠a de un Modelo

    Veamos el modelo User en detalle:

    class User(Base):                    # 1. Hereda de Base
    __tablename__ = "users"          # 2. Nombre de la tabla

    id = Column(Integer, primary_key=True)  # 3. Clave primaria

    1. Herencia de Base

    • Todos los modelos heredan de Base
    • Proporciona funcionalidad ORM autom√°tica
    • Permite crear tablas autom√°ticamente

    2. __tablename__

    • Define el nombre de la tabla en la base de datos
    • Convenci√≥n: plural, snake_case
    • Ejemplos: users, products, order_items

    3. Clave Primaria

    id = Column(Integer, primary_key=True)
    - Integer: Tipo de dato entero - primary_key=True: Identifica √∫nicamente cada registro - Auto-incremento: SQLite lo hace autom√°ticamente

    4. Columnas de Datos

    name = Column(String)
email = Column(String)
price = Column(Float)
quantity = Column(Integer)

    Tipos de Datos Comunes

    Tipo SQLAlchemy Tipo Python Descripci√≥n Ejemplo
    Integer int N√∫meros enteros age = Column(Integer)
    String str Texto variable name = Column(String(100))
    Float float N√∫meros decimales price = Column(Float)
    Boolean bool Verdadero/Falso active = Column(Boolean)
    DateTime datetime Fecha y hora created_at = Column(DateTime)
    Text str Texto largo description = Column(Text)

    Relaciones Entre Modelos

    Aunque nuestro proyecto usa un enfoque simple, es importante entender las relaciones:

    Tipos de Relaciones

    1. Uno a Muchos (One-to-Many)

    from sqlalchemy import ForeignKey
from sqlalchemy.orm import relationship

class User(Base):
    __tablename__ = "users"
    id = Column(Integer, primary_key=True)
    name = Column(String)

    # Relaci√≥n: Un usuario puede tener muchas √≥rdenes
    orders = relationship("Order", back_populates="user")

class Order(Base):
    __tablename__ = "orders"
    id = Column(Integer, primary_key=True)
    user_id = Column(Integer, ForeignKey("users.id"))  # Clave for√°nea

    # Relaci√≥n: Una orden pertenece a un usuario
    user = relationship("User", back_populates="orders")

    2. Muchos a Muchos (Many-to-Many)

    from sqlalchemy import Table

# Tabla de asociaci√≥n

    'user_products',
    Base.metadata,
    Column('user_id', Integer, ForeignKey('users.id')),
    Column('product_id', Integer, ForeignKey('products.id'))

class User(Base):
    __tablename__ = "users"
    id = Column(Integer, primary_key=True)

    # Un usuario puede tener muchos productos favoritos
    favorite_products = relationship(
        "Product",
        secondary=user_product_association,
        back_populates="favorited_by"
    )

class Product(Base):
    __tablename__ = "products"
    id = Column(Integer, primary_key=True)

    # Un producto puede ser favorito de muchos usuarios

        "User",
        secondary=user_product_association,
        back_populates="favorite_products"
    )

    Creaci√≥n de Tablas

    M√©todo Autom√°tico

    En main.py, las tablas se crean autom√°ticamente:

# Crear todas las tablas definidas en los modelos

    ¬øQu√© hace create_all()?

    1. Examina todos los modelos que heredan de Base
    2. Genera el SQL necesario para crear las tablas
    3. Ejecuta las sentencias CREATE TABLE
    4. Ignora tablas que ya existen

    3. Validaci√≥n de Datos

    from sqlalchemy.orm import validates

class User(Base):
    __tablename__ = "users"

    id = Column(Integer, primary_key=True)
    name = Column(String)
    email = Column(String)

    @validates('email')
    def validate_email(self, key, address):
        if '@' not in address:
            raise ValueError("Email inv√°lido")
        return address

    @validates('name')
    def validate_name(self, key, name):
        if len(name) < 2:
            raise ValueError("Nombre muy corto")
        return name
